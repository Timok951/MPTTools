PostgreSQL backup / restore

Для PostgreSQL используйте логический дамп и restore через стандартные утилиты. Это удобно показать на защите как регламент восстановления.

Backup

pg_dump -h {{ db_host|default:"localhost" }} -p {{ db_port|default:"5432" }} -U {{ db_user|default:"postgres" }} -d {{ db_name }} -Fc -f backup.dump

Restore

pg_restore -h {{ db_host|default:"localhost" }} -p {{ db_port|default:"5432" }} -U {{ db_user|default:"postgres" }} -d {{ db_name }} --clean --if-exists backup.dump

После restore выполните проверку: вход в систему, список оборудования, API `/api/v1/schema/`, и одну админскую процедуру из portal.
